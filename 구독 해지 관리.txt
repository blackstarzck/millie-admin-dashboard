문의사항 관리 카테고리 하위에 사용자의 구독 해지와 관련된 내용을 보고 처리할 수 있는 페이지가 필요해. 아래 내용은 이를 수행하기 위한 기획서야.

목적: 사용자 해지 요청 내역 조회, 환불 금액 산출 및 처리, 상태 관리
로직 요약:
1. 자동결제 해지
- 사용자가 구독관리 페이지에서 자동결제 해지를 선택한 경우, 즉시 다음 결제는 중단되며 현재 결제 주기 종료일까지 서비스 이용 가능.
2. 중도 해지
- 사용자가 중도 해지를 선택한 경우, 서비스 이용이 즉시 종료되고 1:1 문의를 통해 환불 절차가 진행됨.
3. 환불 규정
- 결제 후 7일 이내 해지 시 → 전액 환불
- 결제 후 7일 초과 시 → 결제 금액의 9% 환불
- 관리자는 환불 승인 시 금액 자동 계산 및 지급 실행 가능

사용자 플로우 요약
[사용자] 구독관리 페이지
   ↓
자동결제 해지 선택 → 서비스 유지 (만료일까지)
   ↓ 자동반영
관리자(Admin) 페이지 → '자동결제 해지' 상태 표시

또는

중도 해지 선택 → 서비스 즉시 종료
   ↓
사용자 1:1 문의 등록 → 환불 요청 생성
   ↓
관리자(Admin) 페이지 → 환불 요청 확인 / 승인 / 거절 / 처리완료

페이지 구성:
검색조건: 사용자 이름, 이메일, 구독상품명
상태 필터: 전체 / 자동결제 해지 / 중도 해지 / 환불 요청 / 환불 완료
기간 필터: 신청일 기준 (시작일 ~ 종료일)

해지/환불 목록 테이블:
| 항목    | 설명                   |
| ----- | -------------------- |
| 사용자명  | 사용자 이름 또는 닉네임        |
| 이메일   | 계정 이메일               |
| 구독상품명 | 구독 중이던 상품명           |
| 신청유형  | 자동결제 해지 / 중도 해지      |
| 신청일   | YYYY-MM-DD           |
| 결제일   | YYYY-MM-DD           |
| 환불금액  | 자동 계산 (조건에 따라 전액/9%) |
| 상태    | 요청 / 승인 / 거절 / 처리완료  |
| 담당자   | 처리한 관리자명             |
| 액션    | 상세보기 버튼              |

상세보기 모달/페이지:
| 구분    | 항목                                              | 설명 |
| ----- | ----------------------------------------------- | -- |
| 사용자정보 | 이름 / 이메일 / 가입일 / 결제이력 링크                        |    |
| 구독정보  | 상품명 / 결제일 / 결제금액 / 만료일                          |    |
| 신청정보  | 신청유형 / 신청일 / 환불 사유 (사용자 입력)                     |    |
| 환불계산  | 자동 계산: <br>① 결제 후 7일 이내 → 100% <br>② 7일 초과 → 9% |    |
| 관리자처리 | [승인] [거절] [보류] 버튼 및 메모 입력창                      |    |
| 로그    | 상태 변경 이력 (처리자, 날짜, 내용)                          |    |

상태 정의
| 상태   | 설명                   | 전환 가능 상태 |
| ---- | -------------------- | -------- |
| 요청   | 사용자가 해지/환불 신청한 상태    | 승인, 거절   |
| 승인   | 환불금 산출 후 승인된 상태      | 처리완료     |
| 거절   | 관리자에 의해 반려된 상태       | -        |
| 처리완료 | PG사 환불 완료 등 실제 처리 완료 | -        |

예외 및 관리 포인트:
| 상황               | 관리자 조치                   |
| ---------------- | ------------------------ |
| 환불 금액이 계산과 다를 경우 | 수동 입력 가능 (override)      |
| PG사 환불 실패        | 상태를 ‘보류’로 표시하고 재시도 버튼 제공 |
| 동일 계정의 중복 요청     | 최근 요청만 유효하도록 처리          |
| 이용권 만료 전 자동결제 해지 | 서비스 이용기간 종료일 표시 필수       |


로그 및 감사 기록
- 각 환불 건별로 상태 변경 이력 저장 (처리자, 일시, 메모)
- Excel 다운로드 지원 (환불요청내역_YYYYMMDD.xlsx)
